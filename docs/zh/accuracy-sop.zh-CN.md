# 精度提升 SOP（中文）

## 目标

本文档用于指导 `id-doc-ocr` 在证件、登机牌、车票、病例、户口本等文档上的精度提升实践。核心目标不是单纯提高 OCR 字符准确率，而是提高**字段级准确率**和**文档级成功率**。

---

## 1. 核心指标

必须统一追踪以下指标：

- 字段完全正确率（Field Exact Match）
- 关键字段完全正确率
- 文档级成功率（关键字段全部正确）
- 人工复核率（Review Rate）
- 错误放行率（False Accept Rate）
- 延迟与运行成本

### 关键字段示例
- 身份证号
- MRZ
- 姓名
- 票号
- 日期
- 出发地 / 到达地
- 病案号 / 就诊日期

---

## 2. 错误分桶

所有错误都应归因到以下类别之一：

- detect_error：检测/裁切错误
- quality_error：模糊、反光、阴影、遮挡
- ocr_error：识别错误
- layout_error：字段定位错误
- parser_error：字段抽取逻辑错误
- validator_error：校验逻辑错误
- label_error：标注错误
- schema_error：字段定义不合理

优化前先统计错误分布，再决定优先修哪里。

---

## 3. 数据 SOP

### 数据集分层
- golden set：高质量人工确认评测集
- train set：训练集
- val set：验证集
- hard set：难例集（模糊、反光、斜拍、遮挡、新模板、多语言）

### 标注要求
至少包含：
- 区域框 / polygon
- 字段值
- 字段与区域关系
- 质量标签
- 模板版本
- 审核状态

### 标注规范要求
- 每个字段必须有唯一规范
- 日期、姓名、地址、票号等要统一归一化规则
- hard set 建议占优化工作集的 20% 以上

---

## 4. 预处理 SOP

标准预处理流程：
1. 文档检测
2. 四角定位 / 透视矫正
3. 方向归一化
4. 去边框 / 背景裁切
5. 图像质量打分

### 质量评分项
- blur_score
- glare_score
- shadow_score
- crop_integrity_score
- occlusion_score

根据质量评分分流：
- 正常识别
- 增强后识别
- 送人工复核

---

## 5. 抽取策略 SOP

优先级固定为：
1. 固定模板 ROI 抽取
2. 锚点 + 空间关系抽取
3. VLM / KIE fallback

原则：**先确定性，后生成式。**

---

## 6. Parser SOP

每个字段抽取都建议按三阶段设计：
1. 候选召回
2. 候选打分
3. 归一化输出

### 候选召回信号
- 关键词锚点
- 正则命中
- bbox 邻近关系
- 模板 ROI
- 语言/字符集过滤

### 候选打分信号
- 文本匹配度
- 与锚点距离
- 格式合法性
- OCR 置信度
- 文档变体一致性

### 输出归一化
- 日期统一格式
- 编码类字段统一大小写
- 去除多余空格和 OCR 噪声

---

## 7. Validator SOP

校验必须独立存在，至少包括：
- 格式校验
- 长度校验
- checksum / MRZ 校验
- 枚举值校验
- 跨字段一致性校验
- 文档完整性校验

输出建议分为：
- accept
- accept_with_warning
- review
- reject

---

## 8. 人工复核闭环 SOP

低置信度或规则冲突结果必须进入复核。

复核界面至少展示：
- 原图
- 矫正图
- OCR 行结果
- 候选字段
- 最终字段
- 校验报错
- bbox 证据

复核结果必须回流到：
- hard set
- parser regression case
- validator regression case
- 模板更新集

---

## 9. 每周优化节奏

1. 导出错误样本
2. 按根因聚类
3. 选择收益最高的问题
4. 先补测试
5. 做最小修复
6. 跑回归
7. 灰度上线
8. 观察成功率 / 复核率 / 错误放行率

---

## 10. 文档类型建议

### 证件
- 模板路由
- ROI 抽取
- 校验位 / MRZ

### 登机牌 / 车票
- 先做文档变体分类
- 使用关键词锚点 + 空间关系
- 不建议一个 parser 同时硬做所有票据

### 病例
- 先做页面类型分类
- 支持列表字段和关系字段
- 逐步引入医学词典与标准化

### 户口本
- 先做页类型分类
- 做身份证号与家庭关系校验
- 后续支持多页合并
